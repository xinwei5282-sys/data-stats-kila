# data-stats-kila

抖音「圣农」品牌达人投放业绩统计 — 纯本地 HTML 工具。
拖入多个抖店导出的 Excel → 自动输出**达人维度**和**商品维度**两张汇总表 → 一键导出。

## 快速开始

双击 `index.html` 在浏览器打开，把 11 个 Excel 文件拖入页面即可。
零依赖、零安装、所有计算在浏览器本地完成（**不上传任何数据**）。

## 数据源（11 个 Excel）

文件按**文件名关键词**识别，缺哪个就报哪个。

| 关键词 | 类型 | 说明 |
|---|---|---|
| `合作视频` | 合作视频明细 | 抖店导出，视频粒度（达人/商品/投放消耗/支付金额） |
| `精选联盟` | 精选联盟订单明细 | 抖店导出，订单流水（取 `Sheet1`） |
| `达人发布` | 达人发布明细 | 人工录入，谁/何时/发什么 |
| `乘方推广` + `产品` | 乘方推广-产品 | 按商品的广告投放数据 |
| `乘方推广` + `达人` | 乘方推广-达人 | 按达人的广告投放数据 |

「乘方推广」可以是多个账号的多份文件（如 `麦兮`/`麦兮1`/`麦兮2`/`麦兮3`），**自动合并**。

## 关键计算口径

### 达人维度（按 达人 × 商品 展开）

| 字段 | 口径 |
|---|---|
| 合作达人范围 | 乘方推广-达人 4 账号合并去重（只看真正投流的） |
| 当月发布条数（主行） | 合作视频明细表 `发布时间 ∈ 4月` 该达人 count |
| GSV（子行，按商品） | 精选联盟订单 `作者账号 = 该达人 AND 商品id = 该商品` sum(支付金额) |
| 达人合计GSV（主行） | 精选联盟订单 该达人 sum(支付金额)。**自洽：子项 GSV sum = 达人合计 GSV** |
| 当月发布素材投放消耗 | 合作视频明细表 `发布时间 ∈ 当前月` 该达人 sum(投放消耗) |
| 投放消耗（含非当月） | 乘方推广-达人 4 账号合并 `日期='全部'` 该达人 sum(整体消耗) |

### 商品维度（17 个商品来自乘方推广-产品表去重）

| 字段 | 口径 |
|---|---|
| 分类 | **手编**，localStorage 持久化（key `kila-stats:product-meta`） |
| 客单价 | **手编**，localStorage 持久化 |
| 4月素材发布条数 | 达人发布明细表 `月份='4月'` 该商品 count（产品名 → 商品ID 反查） |
| 投放消耗 | 乘方推广-产品 4 账号合并 `日期='全部'` 该商品 sum(整体消耗) |
| 净成交金额 | 乘方推广-产品 4 账号合并 `日期='全部'` 该商品 sum(净成交金额) |
| ROI | 净成交金额 / 投放消耗（除零显示 `-`） |
| 主要产出视频链接 | 合作视频明细表 该商品 按支付金额降序 top 3 |

### 已确定的弃用口径

- ~~GSV 按 流量来源='视频' 筛选~~ → 弃用，改为全流量来源（保证「子项 sum = 达人合计 GSV」自洽）
- ~~发布数用达人发布明细表~~ → 弃用，改用合作视频明细表（人工录入会漏录）
- ~~客单价 = 精选联盟支付金额/商品数量自动算~~ → 弃用，新品 3 个商品无销售数据算不出来

## 数据安全

- Excel 在浏览器**本地**用 SheetJS 解析，不上传任何服务器
- 仓库里**只有 `index.html`**，不会带任何业务数据
- `localStorage` 的手编值只在当前浏览器，换浏览器/清缓存会丢

## 数据源不一致时的处理

- **达人发布明细表** 是人工录入，可能漏录 → 不影响达人维度（已改用合作视频明细表）
- **新品商品**（如薯片炸鸡）在 4 月乘方推广里没数据 → 商品维度会显示 0 投放/0 销售
- **反查失败**：产品名在乘方推广表里找不到 → 控制台 warn，不影响主流程

## 月度更新

下个月（5 月）数据到了：
1. 打开页面
2. 拖入新一批 Excel（文件名前缀可以是「5月」，识别只看关键词）
3. 之前手编的分类/客单价**自动按商品ID 回填**
4. 点「导出 Excel」生成 `kila-数据汇总-YYYYMMDD.xlsx`
