# data-stats-kila 项目进度笔记

## 项目背景
- 用户：蔚然
- 业务：抖音「圣农」品牌达人投放业绩统计（kila 对接）
- 痛点：用户当前用 Excel 公式手工拼汇总表，公式经常报 #DIV/0! 和 #VALUE!，体验差
- 目标：纯本地 HTML 页面，拖拽 Excel → 自动出 2 张汇总表 → 可导出

## 技术选型（已确认）
- 纯本地 HTML（双击即用，零部署、零依赖安装）
- SheetJS (xlsx.full.min.js) 通过 CDN 引入
- 原生 JS（不上框架，体量不需要）
- localStorage 持久化手编字段
- 仅自己用，无需多账号合并扩展（按汇总表注释自然合并多份"乘方推广"表）

## 数据源（位于 /Users/weiran/Desktop/数据源-kila）
1. 4月合作视频明细表.xlsx — 视频维度（10k 行），含达人/商品/投放消耗
2. 4月精选联盟订单明细表.xlsx — 订单流水（93k 行，Sheet1 是主表）
3. 4月达人发布明细表.xlsx — 谁/何时/发什么（1.2k 行）；商品ID 列大量为空，需用产品名反查
4. 4月麦兮1-乘方推广数据表-产品.xlsx — 按商品聚合的广告投放数据
5. 4月麦兮1-乘方推广数据表-达人.xlsx — 按达人聚合的广告投放数据
6. 4月麦兮-乘方推广 / 麦兮2 / 麦兮3 — 全部有数据（之前 Python openpyxl 读取错误，
   误判为空。实际 4 个账号合并后：产品 1316 行 / 达人 1781 行）

## 已确认的口径
- 商品分类：页面让用户手编，不预置字典
- 商品客单价：页面让用户手编
- 不做图表、不做筛选、不做多账号合并扩展

## 待确认的口径（在 task 3 / 4 实现时再问用户）
- 达人维度 "GSV vs 达人合计GSV" 的精确区别（汇总表里两列数值不同）
- "投放消耗（含非当月）" 是否就是乘方推广-达人表的"整体消耗"列
